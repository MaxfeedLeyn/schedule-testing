# Чеклісти для тестування

## Модуль: Управління довідником предметів

### Функціональність
- [ ] Відкривається форма створення предмета.
- [ ] Відкривається форма редагування предмета.
- [ ] Можна зберегти предмет з валідною назвою.
- [ ] Після збереження новий предмет з’являється у списку у алфавітному порядку
- [ ] Після редагування предмета зміни відображаються у списку у алфавітному порядку

### Негативні сценарії
- [ ] Не можна зберегти предмет із порожньою назвою.
- [ ] Не можна зберегти предмет із назвою меншою за мінімально допустиму довжину.
- [ ] Не можна зберегти предмет із назвою більшою за максимальну довжину.
- [ ] Не можна створити предмет із назвою, яка вже існує.
- [ ] Повідомлення про помилку відображається після неуспішного збереження.
- [ ] Дані не зберігаються, якщо натиснути Cancel/Back без підтвердження збереження.

### UI/UX
- [ ] Усі поля мають зрозумілі підписи.
- [ ] Обов’язкові поля позначені як обов’язкові.
- [ ] Кнопка Save неактивна, коли форма невалідна.
- [ ] Текст помилок відображається поруч із відповідним полем.
- [ ] Після помилки курсор/фокус переходить до проблемного поля або помилка легко помітна.
- [ ] На формі немає візуальних зсувів або “ламання” верстки при появі помилок.
- [ ] Інтерфейс однаково зручний для створення і редагування.
- [ ] Довгі назви не ламають вигляд форми або списку.
- [ ] Підказки та кнопки мають достатній контраст і легко читаються.
